# LEO: Unified Real-time Face Detection & Voice Assistant 🤖👁️🎤

Leo is an industry-level, real-time PyQt6 desktop application that seamlessly merges an advanced computer vision **Face Recognition** pipeline with an interactive **Voice Assistant**. 

Designed for smart reception, event management, and secure access, Leo can detect faces, track identities, match them against a local FAISS-indexed database, and generate unique, formal spoken introductions for registered guests using deterministic template logic and TTS.

---

## ✨ Features

### 👁️ Advanced Vision Pipeline
*   **Detection:** Fast and lightweight face localized using `MediaPipe`.
*   **Tracking:** Robust object tracking with `DeepSORT` to maintain stable IDs across frames.
*   **Recognition:** High-accuracy 512-d facial embeddings generated by `ArcFace` (ONNX CPU).
*   **Matching:** Lightning-fast vector search using `FAISS` against an SQLite database.
*   **Stability:** Track voting and cooldown mechanics ensure guests are only greeted once they are confidently identified and prevent spamming.

### 🎙️ Voice & Interactive Assistant
*   **Offline NLU & Speech:** Recognizes natural language intents (Greetings, Playback) via SpeechRecognition.
*   **Custom Welcome Speeches:** A `WelcomeGenerator` dynamically builds ~1-minute, highly formal, unique spoken introductions based on guest profiles (Name, Title, Achievements).
*   **Text-to-Speech (TTS):** Asynchronous TTS worker ensures the camera feed never freezes while Leo is speaking.
*   **Media Playback:** Integrates with `pywhatkit` to play requested media directly from YouTube.

### 🖥️ Modern PyQt6 Dashboard
*   **Live Metrics:** Real-time FPS, Active Tracks, and Known Guests counters.
*   **Guest Enrollment:** A dedicated UI to capture faces directly from the live feed and save them to the database along with designation and achievements.
*   **Welcome Generator Panel:** Select any registered guest to instantly generate, preview, and speak their custom formal introduction.
*   **Event Log:** Real-time logging of system events, recognitions, and voice commands.

## 🚀 Installation & Setup

### 1. Requirements
Ensure you have **Python 3.10 to 3.12** installed. *(Note: Python 3.13 may experience Windows DLL initialization bugs with onnxruntime and PyQt6).*

### 2. Install Dependencies
```powershell
pip install -r requirements.txt
```
*Core packages include:* `PyQt6`, `opencv-python`, `mediapipe`, `onnxruntime`, `faiss-cpu`, `SpeechRecognition`, `pyttsx3`, `numpy`, `Pillow`.

### 3. Model Weights
Download the pre-trained ArcFace ONNX model (e.g., `w600k_r50.onnx` from the InsightFace model zoo) and place it in your configured model directory (`face detection/leo_face/models/`).

## 🎮 Usage

### Launching the App
To start the unified dashboard, run:
```powershell
cd gui_app
python app.py
```

### Guest Enrollment
1. Start the camera using the **Start Camera** button.
2. Ask the guest to look clearly at the camera.
3. In the right panel under **📸 GUEST ENROLLMENT**, fill in their Name, Title, and Achievements.
4. Click **Capture & Enroll Guest**. The embedding will be extracted, saved to SQLite, and the FAISS index will auto-rebuild.

### Generating Welcome Speeches
1. In the **🎙️ WELCOME SPEECH** panel, select a newly enrolled guest from the dropdown.
2. Click **Generate & Speak Welcome**.
3. A formal introduction will be generated using their unique profile data, displayed on the screen, and spoken aloud by the TTS assistant.

### Voice Commands
1. Click **🎤 Start Voice**.
2. Speak clearly into your microphone.
3. Try standard commands like:
    * *"Hello Leo"* -> Triggers introduction response.
    * *"Play [song name]"* -> Opens YouTube and plays the requested video.
    * *"Stop listening"* -> Shuts down the voice assistant.

## 🛠️ Troubleshooting

**Issue:** `ArcFace model not loaded (onnxruntime may not be installed)`  
**Solution:** This is a common C++ DLL initialization conflict on Windows environments when GUI libraries load before ONNX Runtime. In `app.py`, `onnxruntime` is explicitly pre-loaded at line 28 to fix this. If it persists, ensure you have the *Microsoft Visual C++ Redistributable* installed.

**Issue:** `FAISS index unavailable`  
**Solution:** Ensure `faiss-cpu` is correctly installed. You can click the **⚡ Rebuild Index** button in the dashboard's Control panel to force a regeneration from SQLite.
